    Production Deployment

    Overview

    The production system creates two separate versions:

  • Development (this folder): Full CMS editing capabilities

  • Production (deployed): Clean, read-only website

    • Method 1: Automated Deployment

    #### Setup (One-time)

  • Create production repository on GitHub

  • Configure deployment script:

    •  // Edit deploy-production.js, line ~13
     
     this.gitRepoUrl = 'https://github.com/YOUR-USERNAME/YOUR-PRODUCTION-REPO.git';

    #### Deploy

    npm run deploy:production

    This will:

  • Build clean production version

  • Remove all CMS functionality

  • Push to your production GitHub repository

  • Ready for hosting (GitHub Pages, Netlify, etc.)

    • Method 2: Manual Deployment

    #### Step 1: Build Production Version

    npm run build:production

    #### Step 2: Deploy Manually

  • Copy files: Copy everything from production-build/ folder

  • Upload: Upload to your production repository

  • Commit: git add . && git commit -m "Production update"

  • Push: git push origin main

    • What Gets Removed in Production

  • ‚ùå CMS editing interface (/admin folder)

  • ‚ùå Backend APIs (/api folder)

  • ‚ùå Blog management system (/blog-gui-framework)

  • ‚ùå Editable content storage (/content folder)

  • ‚ùå CMS markup from HTML (classes, data attributes)

  • ‚ùå CMS integration scripts

    • What Stays in Production

  • ‚úÖ Clean HTML pages

  • ‚úÖ CSS stylesheets

  • ‚úÖ JavaScript functionality

  • ‚úÖ Images and assets

  • ‚úÖ Static content

    Troubleshooting

    Common Issues

    #### CMS Not Loading

    Problem: CMS interface doesn't appear

    Solution:

    Check if server is running
     
    npm run dev
    Clear browser cache
     
    Open DevTools> Application> Storage> Clear storage

    #### Build Fails

    Problem: npm run build:production fails

    Solutions:

    Check Node.js version (needs 16+)
     
    node --version
    Reinstall dependencies
     
    rm -rf node_modules
     
    npm install
    Check for file permissions
     
    ls -la build-production.js

    #### Deployment Fails

    Problem: npm run deploy:production fails

    Solutions:

  • Check GitHub repository URL in deploy-production.js

  • Verify Git credentials:

    •  git config --global user.name "Your Name"
     
     git config --global user.email "your.email@example.com"

  • Check repository permissions (make sure you have write access)

    • #### Content Not Saving

    Problem: CMS edits don't persist

    Solutions:

  • Check browser storage: DevTools> Application> Local Storage

  • Disable browser extensions that might block storage

  • Try incognito mode to test

    • Development Tips

    #### File Organization

  • Keep images in content/Images/

  • Organize by type: Avatars/, Projects/, Blog/

  • Use descriptive filenames

    • #### Performance

  • Optimize images before uploading

  • Test on mobile devices regularly

  • Check loading speeds with DevTools

    • #### Backup Strategy

    Backup CMS content
     
    Export from browser: DevTools> Application> Local Storage
     
    Copy the cms-content-data entry
    Backup files
     
    cp -r content/ backup-content-$(date +%Y%m%d)/
